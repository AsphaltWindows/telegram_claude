- Forum topic filenames with colons (from ISO-8601 timestamps like `T00:00:00Z`) can trigger suspicious path warnings. Use date-only format (e.g., `2026-03-18`) in filenames instead.
- The idle timer in `session.py` only resets on user input (`send()`), not on agent output. This is a known design gap -- any "agent is unresponsive during long tasks" report likely traces back to `last_activity` not being updated in `_read_stdout()`.
- When a forum topic closes with all votes but the next pipeline step (e.g., ticket creation) depends on an agent acting post-close, verify the message actually entered the pipeline. The handoff from "forum consensus" to "ticket in pipeline" can silently stall.
- The `last_activity` gap in `_read_stdout()` was actually fixed (line 400 now resets it). But the `_typing_heartbeat` still fires every 5s unconditionally during agent silence, which is the real source of "frozen bot" UX complaints. The typing indicator itself works correctly -- the problem is that typing alone is insufficient feedback for long operations.
- The most likely cause of "bot goes silent after agent uses tools" is a protocol-level issue: the bot runs claude with `--print` mode, which may treat tool-use turns as complete (emitting `result` and waiting for new stdin). The bot skips `result` events and never sends a continuation, so the agent blocks forever. Additionally, the `result` event may carry the only copy of the post-tool-use response text. Always check `result` event handling when debugging silent-bot issues.
